Project: MedKnow — healthcare AI knowledge search

A production-grade RAG system that lets healthcare staff search internal protocols, clinical guidelines, and policies using natural language — with verifiable citations, audit logging, and HIPAA-aware architecture. Your healthcare domain experience is the moat. Most engineers building RAG have zero domain credibility. You don't.

Architecture stack
  • Ingestion: PDF/DOCX → text extraction → semantic chunker → embedding API → pgvector (Postgres)
  • Query: Spring Boot API → intent layer → hybrid BM25 + vector search → cross-encoder reranker → LLM with citations
  • Agent: tool-calling agent for multi-hop queries. Tools: search, summarize, fetch-protocol
  • Observability: OpenTelemetry → Prometheus → Grafana. LangFuse for LLM cost + quality tracking
  • Resilience: Resilience4j (circuit breaker, retry, bulkhead), Redis rate limiter, distributed locking
  • Infrastructure: AWS ECS Fargate, RDS Postgres + pgvector, ElastiCache Redis, ALB + HTTPS
  • Auth: JWT + RBAC. Audit log on every query. HIPAA-aware design throughout.

Mistakes to avoid
  • ✕Trying to do 2–3 topics in a single 45-min session. One concept per day compounds better than three concepts half-understood.
  • ✕Skipping the "publish to GitHub" step because it feels like extra work. That step is 30% of the value — it forces you to actually understand what you did.
  • ✕Skipping LinkedIn posts when busy. This is the highest-leverage low-time activity in the plan. 45 minutes of writing compounds for months.
  • ✕Switching the project idea after Day 44. Depth beats novelty. The compounding comes from one thing shipped well, not four things started.
  • ✕Treating Phase 4 as optional. The application/consulting activation step is where the investment pays off. Don't stop at Day 90.
  • ✕Spending catch-up time on reading instead of building. When behind, cut reading first, keep building.
  • ✕Waiting until everything is polished to publish. Publish ugly first, improve publicly. A rough live project beats a perfect local one.
