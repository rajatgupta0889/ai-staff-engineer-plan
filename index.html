Project: MedKnow — Healthcare AI Knowledge Search System
A production-grade RAG system that lets healthcare staff search internal protocols, policies, and clinical guidelines using natural language — with verifiable citations, audit logging, and HIPAA-aware design.
Architecture Overview
  • Ingestion service: PDF/DOCX → chunker → OpenAI/Cohere embeddings → pgvector (Postgres)
  • Query service: Spring Boot API → intent classification → hybrid search (BM25 + vector) → reranker → LLM generation with citations
  • Agent layer: tool-calling agent for multi-hop queries (e.g., "what's the protocol for X in patients with Y?")
  • Observability: OpenTelemetry → Prometheus → Grafana. LangFuse for LLM traces and cost tracking
  • Auth: JWT with role-based access. Audit log for every query (compliance requirement)
  • Infrastructure: AWS ECS (Fargate), RDS Postgres with pgvector, ElastiCache (Redis), ALB

Mistakes to Avoid
  • ✕Switching project ideas after Week 6. The compounding value is in depth, not novelty. Ship the thing.
  • ✕Treating LinkedIn posts as optional. They are half the plan. Engineers who write publicly build 10x the network of those who don't.
  • ✕Consuming too many resources for any one topic. One good book + one lab exercise > three half-read books.
  • ✕Perfecting the project before deploying. Deploy ugly. Polish iteratively. A deployed rough project beats a polished local one.
  • ✕Skipping the documentation step. The GitHub artifact is what clients and recruiters evaluate — not the code you ran locally and forgot.
  • ✕Starting DSA prep out of anxiety. At Staff level, system design depth wins over algorithmic breadth. Don't let leetcode FOMO derail the plan.
  • ✕Keeping the project private. Fear of judgment costs you visibility. Publish early, improve publicly.
When You're Low on Time
